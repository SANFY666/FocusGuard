# 🛡️ FocusGuard

**FocusGuard** — це десктопний застосунок для Windows, створений для тайм-менеджменту, контролю фокусу та відстеження тривалості різних активностей. 

Програма ідеально підходить для балансування між продуктивністю та відпочинком. Наприклад, вона допоможе сфокусуватися під час написання об'ємної курсової роботи чи виконання інженерних лабораторних, а потім проконтролює, щоб ваша перерва на матч у Dota 2 чи кілька каток у Brawl Stars не затягнулася на весь вечір. Застосунок дозволяє встановлювати запланований час для активності та отримувати стильні сповіщення, коли час вичерпано.

---

## ✨ Головні можливості

* **⏱ Відстеження сесій:** Точний підрахунок часу, витраченого на активність (запланований час vs. фактичний час).
* **🔔 Кастомні сповіщення:** Власна система діалогових вікон (`CustomAlertForm`), яка замінює стандартні системні повідомлення Windows на стильні попапи.
* **🎨 Сучасний UI/UX:** Темна тема (Dark Mode), плоский дизайн (Flat Design) та плавні заокруглення інтерфейсу замість застарілих прямокутних форм.

---

## ⚙️ Архітектура та Технології

Проєкт побудований на мові **C# (фреймворк .NET Windows Forms)**. Використовуються принципи **Event-Driven Architecture** (Архітектура, керована подіями) та підхід **Separation of Concerns** (Розділення відповідальності), що відокремлює візуальне відображення від математичної логіки підрахунку часу.

### 🏗 Структура компонентів
Застосунок логічно розділений на три основні модулі:

1. **Presentation Layer (Користувацький інтерфейс):**
   * `Form1.cs` — Головне вікно, яке відповідає за ініціалізацію програми, прийом даних від користувача (наприклад, введення запланованого часу сесії) та управління станами таймерів.
   * `CustomAlertForm.cs` — Динамічне вікно сповіщення. Створюється в пам'яті "на льоту", приймаючи параметри (текст повідомлення, колір теми) через конструктор, і автоматично центрує свої елементи незалежно від розміру тексту чи роздільної здатності.

2. **Custom UI Controls (Власні елементи керування):**
   * `RoundedPanel.cs` — Перевизначений компонент на базі стандартного `Panel`. Використовує бібліотеку `System.Drawing.Drawing2D` (зокрема `GraphicsPath`) для малювання згладжених кутів. Це дозволяє обійти візуальні обмеження класичного WinForms та реалізувати сучасний інтерфейс.

3. **Business Logic Layer (Бізнес-логіка):**
   * `TimeSession.cs` — Ядро системи трекінгу. Інкапсулює всю логіку роботи з часом: фіксує момент старту, розраховує поточну тривалість сесії за допомогою об'єктів `TimeSpan` та перевіряє, чи досягнуто встановленого ліміту. Відв'язка цієї логіки від форми робить код чистішим та відкритим до подальшого масштабування.

### 🔄 Життєвий цикл сесії (Data Flow)
1. **Ініціалізація:** Користувач задає параметри активності та запускає сесію.
2. **Трекінг:** Створюється екземпляр `TimeSession`. Об'єкт `Timer` генерує події (ticks) щосекунди, оновлюючи інтерфейс користувача актуальними даними.
3. **Валідація:** Під час кожного "тіку" бізнес-логіка звіряє фактичний час із запланованим.
4. **Завершення:** При досягненні ліміту фоновий таймер зупиняється, трекінг припиняється, і система викликає екземпляр `CustomAlertForm`, м'яко перериваючи активність користувача.
